# Collectors

Registry collection reads HKCU/HKLM Run and RunOnce through 32/64-bit views. Services read automatic-start configuration from the read-only Services Registry key; current service state is unavailable in this MVP. Scheduled tasks use the official Task Scheduler 2.0 `Schedule.Service` COM API to recursively enumerate folders and registered tasks, including hidden tasks. The collector reads task path, enabled state, principal, run level, Exec actions, and trigger summaries from the API-supplied definition XML with DTD and external resolution disabled. It does not use `C:\Windows\System32\Tasks` as a collection source, execute a task, or modify Task Scheduler. Protected folders/tasks may still yield individually structured partial errors. Startup folders do not follow reparse points; `.lnk` files are retained but their targets, arguments, and working directories are not resolved in this MVP.
